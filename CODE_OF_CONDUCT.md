# Code of Conduct

## Purpose

CXF is a standards and interoperability project. Productive discussion requires clarity, rigor, respect, and good-faith collaboration.

## Expected behavior

Participants are expected to:

- discuss technical issues in good faith
- critique ideas, not people
- provide evidence, examples, and concrete reasoning
- be respectful across institutions, jurisdictions, and experience levels
- accept correction when facts, standards, or test results disagree
- help keep discussions focused, reviewable, and technically useful

## Unacceptable behavior

The following behavior is not acceptable:

- harassment or intimidation
- personal attacks or insults
- discriminatory language
- repeated bad-faith argumentation
- deliberate derailment of technical review
- knowingly misrepresenting test results, sources, or interoperability evidence
- publishing private information without permission
- spamming or abusive contact behavior

## Scope

This code applies to:

- GitHub issues
- GitHub discussions
- pull requests
- mailing list communication
- interoperability collaboration spaces
- other official project channels

## Enforcement

Project maintainers may take action including:

- requesting changes in behavior
- moderating or closing discussions
- removing comments
- limiting participation
- escalating severe cases to project governance contacts

## Reporting

Please report conduct concerns to:

- `contact@cxf-standard.org`

For security-sensitive matters, use:

- `security@cxf-standard.org`
